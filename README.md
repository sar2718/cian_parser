# 🏠 Cian Parser

Парсер объявлений о продаже недвижимости с сайта [Cian.ru](https://www.cian.ru/) с возможностью анализа и визуализации собранных данных.

## 📋 Описание

Проект позволяет автоматически собирать детальную информацию об объявлениях о продаже квартир на Cian.ru и проводить их последующий анализ. Парсер обходит ограничения пагинации через разбиение на ценовые диапазоны и использует задержки между запросами для предотвращения блокировок.

### Собираемые данные

- **Основная информация**: заголовок, цена, количество комнат
- **Местоположение**: адрес, координаты (широта/долгота)
- **Транспортная доступность**: ближайшие станции метро и время до них
- **Характеристики квартиры**: площадь, этаж, состояние и другие параметры
- **Характеристики дома**: год постройки, тип дома, материал стен
- **Описание**: полный текст объявления

## ⚙️ Настройка

Все настройки находятся в файле `config.py`:

```python
# Основные параметры поиска
DEAL_TYPE = "sale"          # Тип сделки: "sale" (продажа) или "rent" (аренда)
ROOMS = 1                   # Количество комнат
REGION = 1                  # Регион (1 = Москва)

# Диапазоны цен для парсинга
LIMITS = tuple(range(10_000_000, 250_000_000 + 1, 1_000_000))

# Задержки между запросами (в секундах)
DELAY_BETWEEN_PAGES = (1, 3)    # Между страницами списка
DELAY_BETWEEN_ADS = (1, 2)      # Между объявлениями

# Файл для сохранения результатов
OUTPUT_DIRNAME = "trash"
OUTPUT_FILENAME = "test"
```

## 📖 Использование

### Запуск парсера

```bash
python main.py
```

Парсер начнет собирать данные согласно настройкам в `config.py` и сохранит результаты в JSON-файл.

## 📊 Анализ данных

В папке `analyse/` находятся скрипты для анализа собранных данных:

- **`import_json.py`** - импорт данных из JSON
- **`remove_dublicates.py`** - удаление дубликатов
- **`load_to_ch.py`** - загрузка данных в ClickHouse
- **`histogram.py`** - построение гистограмм распределения цен
- **`boxplot.py`** - боксплоты для анализа выбросов
- **`heatmap.py`** - тепловая карта объявлений на географической карте
- **`district.py`** - анализ по районам

## 🛡️ Особенности реализации

### Обход ограничений

- **Разбиение на ценовые диапазоны**: Cian ограничивает пагинацию, поэтому парсер разбивает поиск на множество узких ценовых диапазонов
- **Ротация User-Agent**: используется пул различных User-Agent для имитации разных браузеров
- **Задержки между запросами**: случайные паузы предотвращают блокировку по частоте запросов
- **Повторные попытки**: автоматический retry при сетевых ошибках и HTTP 429

### Надежность

- Обработка ошибок на каждом этапе
- Логирование прогресса парсинга
- Сохранение данных в процессе работы (не требуется ждать завершения всего парсинга)

## ⚠️ Важные замечания

1. **Соблюдайте robots.txt** и условия использования сайта Cian.ru
2. **Не злоупотребляйте частотой запросов** - используйте разумные задержки
3. **Используйте данные ответственно** - только для личных, образовательных или исследовательских целей
4. Структура HTML-страниц может измениться, что потребует обновления селекторов
